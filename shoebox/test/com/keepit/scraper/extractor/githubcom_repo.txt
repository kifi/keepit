          <span class="name"><span class="mini-icon mini-icon-readme"></span> README.md</span><article class="markdown-body entry-content" itemprop="mainContentOfPage"><h3>
<a name="fabric-" class="anchor" href="#fabric-"><span class="mini-icon mini-icon-link"></span></a>Fabric <a href="http://travis-ci.org/#!/kangax/fabric.js"><img src="https://secure.travis-ci.org/kangax/fabric.js.png?branch=master" alt="Build Status" style="max-width:100%;"></a>
</h3>

<p><strong>Fabric.js</strong> is a framework that makes it easy to work with HTML5 canvas element. It is an <strong>interactive object model</strong> on top of canvas element. It is also an <strong>SVG-to-canvas parser</strong>.</p>

<p><a href="http://fabricjs.com/kitchensink/" target="_blank"><img src="https://github.com/kangax/fabric.js/raw/master/lib/screenshot.png" style="max-width:100%;"></a></p>

<p>Using Fabric.js, you can create and populate objects on canvas; objects like simple geometrical shapes — rectangles, circles, ellipses, polygons, or more complex shapes consisting of hundreds or thousands of simple paths. You can then scale, move, and rotate these objects with the mouse; modify their properties — color, transparency, z-index, etc. You can also manipulate these objects altogether — grouping them with a simple mouse selection.</p>

<p>Contributions are very much welcome!</p>

<h3>
<a name="goals" class="anchor" href="#goals"><span class="mini-icon mini-icon-link"></span></a>Goals</h3>

<ul>
<li>Unit tested (1400+ tests at the moment)</li>
<li>Modular (~20 small "classes" and modules)</li>
<li>Cross-browser</li>
<li><a href="https://github.com/kangax/fabric.js/wiki/Focus-on-speed">Fast</a></li>
<li>Encapsulated in one object</li>
<li>No browser sniffing for critical functionality</li>
<li>Runs under ES5 strict mode</li>
<li>Runs on a server under <a href="http://nodejs.org/">Node.js</a>
</li>
</ul><h3>
<a name="supported-browsers" class="anchor" href="#supported-browsers"><span class="mini-icon mini-icon-link"></span></a>Supported browsers</h3>

<ul>
<li>Firefox 2+</li>
<li>Safari 3+</li>
<li>Opera 9.64+</li>
<li>Chrome (all versions should work)</li>
<li>IE9+</li>
</ul><h4>
<a name="with-help-of-explorer-canvas" class="anchor" href="#with-help-of-explorer-canvas"><span class="mini-icon mini-icon-link"></span></a>With help of <a href="http://code.google.com/p/explorercanvas/">Explorer Canvas</a>
</h4>

<ul>
<li>IE8 (incomplete — about 17 failing tests at the moment)</li>
<li>IE7,6 (incomplete - about 27 failing tests at the moment)</li>
</ul><p>You can <a href="http://kangax.github.com/fabric.js/test/unit/">run automated unit tests</a> right in the browser.</p>

<h3>
<a name="history" class="anchor" href="#history"><span class="mini-icon mini-icon-link"></span></a>History</h3>

<p>Fabric.js started as a foundation for design editor on <a href="http://printio.ru">printio.ru</a> — interactive online store with ability to create your own designs. The idea was to create <a href="http://printio.ru/ringer_man_tees/new">Javascript-based editor</a>, which would make it easy to manipulate vector shapes and images on T-Shirts. Since performance was one of the most critical requirements, we chose canvas over SVG. While SVG is excellent with static shapes, it's not as performant as canvas when it comes to dynamic manipulation of objects (movement, scaling, rotation, etc.). Fabric.js was heavily inspired by <a href="http://www.ernestdelgado.com/public-tests/canvasphoto/demo/canvas.html">Ernest Delgado's canvas experiment</a>. In fact, code from Ernest's experiment was the foundation of an entire framework. Later, Fabric.js grew into a collection of distinct object types and got an SVG-to-canvas parser.</p>

<h3>
<a name="building" class="anchor" href="#building"><span class="mini-icon mini-icon-link"></span></a>Building</h3>

<ol>
<li><p><a href="https://github.com/joyent/node/wiki/Installation">Install Node.js</a></p></li>
<li>
<p>Build distribution file  <strong>[~76K minified, ~22K gzipped]</strong></p>

<pre><code>$ node build.js
</code></pre>
</li>
</ol><pre><code>- Or build a custom distribution file, by passing (comma separated) module names to be included.&lt;br&gt;

        $ node build.js modules=text,serialization,parser
        // or
        $ node build.js modules=text
        // or
        $ node build.js modules=parser,text
        // etc.

  By default (when none of the modules are specified) only basic functionality is included.
  See the list of modules below for more information on each one of them.
  Note that default distribution has support for **static canvases** only.

  To get minimal distribution with interactivity, make sure to include corresponding module:

        $ node build.js modules=interaction

- You can also include all modules like so:

        $ node build.js modules=ALL
</code></pre>

<ol>
<li>
<p>Create a minified distribution file</p>

<pre><code># Using YUICompressor (default option)
$ node build.js modules=... minifier=yui

# or Google Closure Compiler
$ node build.js modules=... minifier=closure
</code></pre>
</li>
<li>
<p>Optionally, you can build documentation</p>

<pre><code>$ node build_docs.js
</code></pre>
</li>
</ol><h3>
<a name="demos" class="anchor" href="#demos"><span class="mini-icon mini-icon-link"></span></a>Demos</h3>

<ul>
<li><a href="http://fabricjs.com/demos/">Demos</a></li>
<li><a href="http://fabricjs.com/kitchensink/">Kitchensink demo</a></li>
<li><a href="http://fabricjs.com/benchmarks/">Benchmarks</a></li>
</ul><h3>
<a name="documentation" class="anchor" href="#documentation"><span class="mini-icon mini-icon-link"></span></a>Documentation</h3>

<p>Documentation is always available at <a href="http://fabricjs.com/docs/">http://fabricjs.com/docs/</a>. You can also build it locally, following step 4 from the "Building" section of this README.</p>

<p>Also see <a href="http://www.slideshare.net/kangax/fabricjs-building-acanvaslibrarybk">presentation from BK.js</a> and <a href="http://www.slideshare.net/kangax/fabric-falsy-values-8067834">presentation from Falsy Values</a> for an overview of fabric.js, how it works, and its features.</p>

<h3>
<a name="optional-modules" class="anchor" href="#optional-modules"><span class="mini-icon mini-icon-link"></span></a>Optional modules</h3>

<p>These are the optional modules that could be specified for inclusion, when building custom version of fabric:</p>

<ul>
<li>
<strong>text</strong> — Adds support for <code>fabric.Text</code>
</li>
<li>
<strong>serialization</strong> — Adds support for <code>loadFromJSON</code>, <code>loadFromDatalessJSON</code>, and <code>clone</code> methods on <code>fabric.Canvas</code>
</li>
<li>
<strong>interaction</strong> — Adds support for interactive features of fabric — selecting/transforming objects/groups via mouse/touch devices.</li>
<li>
<strong>parser</strong> — Adds support for <code>fabric.parseSVGDocument</code>, <code>fabric.loadSVGFromURL</code>, and <code>fabric.loadSVGFromString</code>
</li>
<li>
<strong>image_filters</strong> — Adds support for image filters, such as grayscale of white removal.</li>
<li>
<strong>easing</strong> - Adds support for animation easing functions</li>
<li>
<strong>node</strong> — Adds support for running fabric under node.js, with help of <a href="https://github.com/tmpvar/jsdom">jsdom</a> and <a href="https://github.com/learnboost/node-canvas">node-canvas</a> libraries.</li>
<li>
<strong>freedrawing</strong> - Adds support for free drawing</li>
</ul><h3>
<a name="examples-of-use" class="anchor" href="#examples-of-use"><span class="mini-icon mini-icon-link"></span></a>Examples of use</h3>

<h4>
<a name="adding-red-rectangle-to-canvas" class="anchor" href="#adding-red-rectangle-to-canvas"><span class="mini-icon mini-icon-link"></span></a>Adding red rectangle to canvas</h4>

<pre><code>&lt;canvas id="canvas" width="300" height="300"&gt;&lt;/canvas&gt;
...
var canvas = new fabric.Canvas('canvas');

var rect = new fabric.Rect({
  top: 100,
  left: 100,
  width: 60,
  height: 70,
  fill: 'red'
});

canvas.add(rect);
</code></pre>

  <h2>Markdown Cheat Sheet</h2>

  <div class="cheatsheet-content">

  <div class="mod">
    <div class="col">
      <h3>Format Text</h3>
      <p>Headers</p>
      <pre>
# This is an &lt;h1&gt; tag
## This is an &lt;h2&gt; tag
###### This is an &lt;h6&gt; tag</pre>
     <p>Text styles</p>
     <pre>
*This text will be italic*
_This will also be italic_
**This text will be bold**
__This will also be bold__

*You **can** combine them*
</pre>
    </div>
    <div class="col">
      <h3>Lists</h3>
      <p>Unordered</p>
      <pre>
* Item 1
* Item 2
  * Item 2a
  * Item 2b</pre>
     <p>Ordered</p>
     <pre>
1. Item 1
2. Item 2
3. Item 3
   * Item 3a
   * Item 3b</pre>
    </div>
    <div class="col">
      <h3>Miscellaneous</h3>
      <p>Images</p>
      <pre>
![GitHub Logo](/images/logo.png)
Format: ![Alt Text](url)
</pre>
     <p>Links</p>
     <pre>
http://github.com - automatic!
[GitHub](http://github.com)</pre>
<p>Blockquotes</p>
     <pre>
As Kanye West said:

> We're living the future so
> the present is our past.
</pre>
    </div>
  </div>
  <div class="rule"></div>

  <h3>Code Examples in Markdown</h3>
  <div class="col">
      <p>Syntax highlighting with <a href="http://github.github.com/github-flavored-markdown/" title="GitHub Flavored Markdown" target="_blank">GFM</a></p>
      <pre>
```javascript
function fancyAlert(arg) {
  if(arg) {
    $.facebox({div:'#foo'})
  }
}
```</pre>
    </div>
    <div class="col">
      <p>Or, indent your code 4 spaces</p>
      <pre>
Here is a Python code example
without syntax highlighting:

    def foo:
      if not bar:
        return true</pre>
    </div>
    <div class="col">
      <p>Inline code for comments</p>
      <pre>
I think you should use an
`&lt;addr&gt;` element here instead.</pre>
